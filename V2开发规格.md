## V2 开发规划

### P0 — 必须完成（v1 遗留 + 核心体验）

#### 1. 对话历史持久化与会话管理
**问题：** 当前版本刷新页面后对话历史丢失，无法查看过往对话。
**方案：**
- 前端加载已存在对话列表（GET /api/conversations）
- 侧边对话列表栏（桌面常驻、手机抽屉）
- 自动恢复上次选中的对话
- 新建对话按钮
- 切换剧本时保留旧对话、自动加载新剧本对话
- 对话删除功能
**工时：** 2 周

#### 2. Service Worker / PWA 重写
**问题：** v1 SW 拦截请求导致页面崩溃，已禁用。
**方案：**
- 重写 SW：纯网络优先策略，不做导航缓存
- 仅缓存 _next/static 资源（CSS/JS）
- 离线页提示
- 添加到主屏幕（manifest.json）
- 充分测试后再启用
**工时：** 1 周

#### 3. 统一认证中间件
**问题：** v1 为了快速跑通，大部分 API 去掉了鉴权。
**方案：**
- 恢复所有 API 的鉴权检查
- 创建 Next.js Middleware 统一注入用户上下文
- 修复前端 x-user-data header 传递（从 React state 读取而非 localStorage）
- Session 从内存 Map 迁移到 Redis（Docker 增加 redis 容器）
- 登录支持 HTTPOnly Cookie + JWT 双因子
**工时：** 1.5 周

### P1 — 应该完成（效率与体验）

#### 5. 改编版本回退
**问题：** 改编后无法恢复，误操作不可逆。
**方案：**
- scripts 表已有 version 字段
- 改编时保存完整快照（before_snapshot）
- 剧本详情页增加版本历史面板
- 一键恢复到任意历史版本
**工时：** 1 周

#### 6. 高频问答缓存
**问题：** 同一剧本重复问题反复调用 LLM，浪费 Token。
**方案：**
- qa_cache 表：script_id + question_hash → answer
- 提问时先查缓存，命中直接返回
- 缓存 24h 过期，剧本更新自动失效
- 预计节省 40-50% API 调用
**工时：** 0.5 周

#### 7. 问答引用溯源
**问题：** AI 回答引用"【第三幕·角色B】"但无法跳转原文。
**方案：**
- 解析时给每段标注位置索引
- Prompt 要求 AI 附带位置标记
- 前端引用变为可点击链接
- 点击弹出原文上下文侧面板
**工时：** 1.5 周

#### 8. 剧本概览卡片 + 快捷提问
**问题：** 新员工面对新剧本不知能问什么。
**方案：**
- 选剧本后显示概览卡片（类型/人数/难度/角色列表/每幕概要）
- 快捷提问按钮（"谁是凶手？""关键线索？""DM开场白？"等）
- 概要由 AI 预生成并缓存
**工时：** 1.5 周

### P2 — 锦上添花

#### 9. 流式输出断点续传
**方案：** SSE 中断后记录 token offset，重连时从断点续传。
**工时：** 1 周

#### 10. 剧本对比阅读模式
**方案：** 改编页原版/改编版左右同步滚动 + 差异高亮。
**工时：** 1 周

#### 11. 扫描版 PDF OCR 支持
**方案：** FastAPI 集成 PaddleOCR，文本<100字时自动切 OCR。
**工时：** 1.5 周

#### 12. 组长角色权限
**方案：** 新增 leader 角色，可看组内排班但不能改系统设置。
**工时：** 1 周

#### 13. 数据仪表盘
**方案：** 管理员首页显示问答次数/成本/热门剧本/员工活跃度。
**工时：** 1.5 周

#### 14. AI 自动排班建议
**方案：** LLM 基于历史排班+员工偏好+预约场次生成排班建议，管理员审核采纳。
**工时：** 1.5 周
