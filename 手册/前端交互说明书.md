## 整体架构


**认证层**: 所有 `/api/*` 请求先经过 middleware JWT 验证，未登录返回 401
**数据刷新**: 全页面 `visibilitychange` 自动刷新

## 跨页面通用规则

### 认证流程 (v2.2)
- 登录时设置 HTTPOnly Cookie (`session_token`)，浏览器自动携带
- 未登录访问任何 API → 401 + 中文提示
- JWT 7天过期，过期后需重新登录
- 登出时清除 Cookie

### 按钮禁用（统一规范）
- 操作中: `disabled={busy/saving/uploading/deleting.has(id)}`
- 视觉: `disabled:bg-gray-400` / `disabled:opacity-50` / `disabled:cursor-not-allowed`
- 文案: "保存中..."/"删除中..."/"上传中..."/"AI分析中..."

### 错误处理
- API错误: 红色内联横幅（不再使用alert）
- 401错误: 清除登录状态，显示登录框
- 删除确认: `confirm()` 原生对话框
- 网络异常: 具体错误信息

## 页面一：主页 `/` — 对话

### 状态变量 (22个)
`user`, `scripts`, `scriptsLoading`, `selectedScript`, `messages`, `input`, `isLoading`, `error`, `search`, `conversationId`, `showScriptPicker`, `scriptSearch`, `historyLoading`, `characterName`, `characterList`, `searchResults`, `searching`, `showSearchResults`

### 关键交互

**选剧本流程:**
```
点"+ 选择剧本" → 弹窗（搜索/类型标签过滤/loading态/空态）
→ 选剧本 → "加载对话历史..."
→ GET /api/conversations → 有对话? → GET /api/chat/history → 加载消息
                      → 无对话? → POST /api/conversations → 创建
→ 异步加载角色列表
```

**搜索对话历史 (v2.2 新增):**
```
输入关键词(≥2字符) → 防抖300ms → GET /api/chat/search?q=xxx&scriptId=xxx
→ 下拉结果（剧本名标签 + 对话标题 + 关键词高亮片段）
→ 点击结果 → 切换到对应对话（跨剧本自动切换）
→ 点击外部或清空输入 → 关闭下拉
```

搜索流程细节：
- 调用 PostgreSQL ILIKE 模糊匹配（支持中英文）
- 返回最多20条，按时间倒序
- 关键词在片段中用 `<mark>` 高亮
- 不传 scriptId 时跨剧本搜索
- 搜索框失焦或点击外部自动关闭结果

**发送消息:**
```
Enter/点发送 → 校验(无剧本/无对话/空输入)
→ POST /api/chat (SSE) {scriptId, conversationId, message, characterName?}
→ 流式拼接 + ●●●动画 → [DONE]
→ 首条无历史问答自动缓存（见QA缓存）
```

**QA缓存机制 (v2.2):**
- 仅缓存"全新对话的第一条问题"
- 缓存24小时，命中时瞬间返回（不走DeepSeek）
- Response Header: `X-QA-Cache: hit`
- 剧本上传后自动清除该剧本的所有缓存

### 各状态
| 状态 | 表现 |
|------|------|
| 未登录 | 居中登录卡片 |
| 剧本列表加载中 | 弹窗内"加载中..." |
| 空剧本列表 | "暂无可选剧本" |
| 对话历史加载中 | "加载对话历史..." |
| AI回复中 | "回答中..." + ●●●弹跳 + textarea禁用 |
| 搜索中 | 搜索框右侧"搜索中..." |
| 搜索无结果 | 下拉显示"无匹配结果" |
| 缓存命中 | 瞬间返回（X-QA-Cache: hit） |

## 页面二：剧本管理 `/scripts`

### 上传弹窗完整交互

**添加文件:**
- 拖拽文件/文件夹 → 递归遍历子目录
- "选择文件" / "选择文件夹" 按钮
- >500MB 或格式不支持 → 自动标记 error

**解析与OCR:**
- 所有文件走 FastAPI `/parse`
- DOCX/PDF(文字版): 秒级
- PDF(扫描版): 自动逐页 OCR，10分钟超时
- PNG/JPG: RapidOCR 识别，原图留存
- 3路并发，进度条实时更新

**去重**: MD5三级去重（批次内→数据库→跨批次）

| 状态 | 表现 |
|------|------|
| 上传中 | 进度条 + 按钮"处理中 (3/6)" + 全表单禁用 |
| OCR解析中 | 进度条缓慢推进（大PDF逐页OCR） |
| 全部成功 | "全部就绪" → 1s后自动关闭 |
| 部分失败 | 红色"X个文件失败，可移除后重新添加" |
