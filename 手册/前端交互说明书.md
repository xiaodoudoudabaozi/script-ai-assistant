## 跨页面通用规则

### 按钮禁用（统一规范）
- 操作中: `disabled={busy/saving/uploading/deleting.has(id)}`
- 视觉: `disabled:bg-gray-400` / `disabled:opacity-50` / `disabled:cursor-not-allowed`
- 文案: "保存中..."/"删除中..."/"上传中..."/"AI分析中..."
- 表单联动: 提交期间所有输入框同步禁用 (`disabled:bg-gray-100`)

### 错误处理
- API错误: 红色内联横幅（不再使用alert）
- 删除确认: `confirm()` 原生对话框
- 提交中禁止关闭: 处理中时取消按钮禁用
- 网络异常: 具体错误信息

## 页面一：主页 `/` — 对话

### 状态变量 (18个)
`user`, `scripts`, `scriptsLoading`, `selectedScript`, `messages`, `input`, `isLoading`, `error`, `search`, `conversationId`, `showScriptPicker`, `scriptSearch`, `historyLoading`, `characterName`, `characterList`

### 关键交互

**选剧本流程:**
```
点"+ 选择剧本" → 弹窗（搜索/类型标签过滤/loading态/空态）
→ 选剧本 → "加载对话历史..."
→ GET /api/conversations → 有对话? → GET /api/chat/history → 加载消息
                      → 无对话? → POST /api/conversations → 创建
→ 异步加载角色列表(GET /api/scripts/files → 提取character_name去重)
```

**角色过滤:**
- 剧本标签栏右侧显示角色下拉选择器（仅当有角色时）
- 选项: "全部角色" + 各角色名
- 选角色后显示"仅看{name}视角"
- 发消息时 `characterName` 传给 `/api/chat`，服务端过滤其他角色剧本

**发送消息:**
```
Enter/点发送 → 校验(无剧本/无对话/空输入)
→ POST /api/chat (SSE) {scriptId, conversationId, message, characterName?}
→ 流式拼接 + ●●●动画 → [DONE]
```

**新对话按钮:**
- 右侧"+ 新对话"按钮
- 点击后清空消息、创建新对话线程
- 旧对话保留在数据库中，切换剧本时自动恢复最新对话

### 各状态
| 状态 | 表现 |
|------|------|
| 未登录 | 居中登录卡片（loading时全禁用） |
| 剧本列表加载中 | 弹窗内"加载中..." |
| 空剧本列表 | "暂无可选剧本" |
| 对话历史加载中 | "加载对话历史..." |
| AI回复中 | "回答中..." + ●●●弹跳 + textarea禁用 |
| 角色过滤激活 | 下拉+紫色标签"仅看{name}视角" |

## 页面二：剧本管理 `/scripts`

### FileEntry 结构
```typescript
{ id, file: File, relativePath, fileType, characterName, status, error }
// status: "pending"|"parsing"|"done"|"error"|"deduped"
```

### 文件类型下拉 (7种)
DM手册 / 主剧本 / 角色剧本 / 线索卡 / 图片线索 / 结局/返场 / 其他

### 上传弹窗完整交互

**1. 添加文件:**
- 拖拽文件/文件夹到虚线区域 → 递归遍历子目录
- 点击"选择文件" → 多文件选择器 (accept: .docx/.doc/.pdf/.png/.jpg/.jpeg)
- 点击"选择文件夹" → 文件夹选择器 (webkitdirectory)
- 大文件: >500MB自动标记error
- 格式不支持: 自动标记error
- 每个文件行: ○状态图标 + 文件名(截断) + 大小 + 类型下拉 + 角色名输入(仅角色剧本) + ×移除

**2. AI自动分类:**
```
点"🤖 AI自动分类" → 读每文件名 → POST /api/scripts/classify
→ 返回 [{fileType, characterName}] → 自动填入类型+角色名
→ 用户可修正 → 保存
```

**3. 解析与OCR:**
- 所有文件统一走 FastAPI `/parse`
- DOCX/PDF(文字版): 直接提取文字，秒级
- PDF(扫描版): 自动触发逐页 OCR，每页约1-3秒，10分钟超时
- PNG/JPG: RapidOCR 识别中英文，同时保存原图到 `/data/scripts/{id}/images/`
- 3路并发解析，进度条实时更新

**4. 去重显示:**
- MD5三级去重：批次内→数据库→跨批次
- 重复文件 → 状态 "≣" + 蓝色"内容重复已合并"

**5. 进度追踪:**
- 进度条: (done+error)/total 百分比
- 每文件状态: ○待处理 → ⟳解析中 → ✓完成 / ≣去重 / ✗失败
- 全部成功→1s后自动关闭弹窗
- 部分失败→红色提示"X个失败，可移除后重新添加或直接保存"

**6. 编辑模式:**
- 打开已有剧本 → 加载已有文件列表(GET /api/scripts/files)
- 显示已有文件: 文件名 + 类型标签 + 角色名 + ×删除按钮
- 可追加新文件(拖入/选择)

### 各状态
| 状态 | 表现 |
|------|------|
| 加载中 | "加载中..." |
| 空列表 | "暂无剧本，点击'+ 添加剧本'开始录入" |
| 上传中 | 进度条 + 按钮"处理中 (3/6)" + 全表单禁用 |
| AI分类中 | 按钮禁用 + 全表单禁用 |
| OCR解析中 | 进度条缓慢推进（大PDF逐页OCR） |
| 全部成功 | "全部就绪，请点击保存" → 1s后自动关闭 |
| 部分失败 | 红色"X个文件失败，可移除后重新添加" |

## 页面三：改编工坊 `/adapt`

### 改编两步流程
```
Step1: 填表单 → "生成改编方案" → POST /api/adaptations {step:"preview"}
→ 敏感本警告 / 禁止指令错误 / 黄色方案预览

Step2: "确认并生成全本" → POST /api/adaptations {step:"generate"}
→ 截断警告(max_tokens 80000, finish_reason检测) / 结果展示

导出: "导出DOCX" → POST /api/adaptations/export → 浏览器下载
对比: "对比原版" → GET缓存 → 左右双栏并排
```

### 4种改编类型
- 元素替换: 改细节保持结构（刀杀→毒杀）
- 剧情微调: 改逻辑衔接（冲突提前/线索位移）
- 视角扩展: 以另一角色视角重写
- 手册改编: 改DM手册措辞格式

### 各状态
| 状态 | 表现 |
|------|------|
| 生成中 | "AI分析中..."/"AI改编中..." + 全表单禁用 |
| 预览就绪 | 黄色背景块 + 方案文字 |
| 截断 | 红色"⚠ 改编内容可能被截断" |
| 对比模式 | 左灰(原版)右紫(改编版) |
| 导出中 | "导出中..." |
