## 四、认证架构（v2.2 重构）

### JWT 认证流程
```
登录 → POST /api/auth/login（白名单）
     → bcrypt验证密码
     → jose签发JWT（HS256, 7天过期）
     → HTTPOnly Cookie + JSON响应双通道

API请求 → middleware.ts 拦截所有 /api/*
        → 读取 session_token cookie
        → jose验证JWT签名+过期
        → 注入 x-user-id / x-user-role header
        → 路由处理器从 header 读取用户信息
```

### 保护范围
- **白名单**: `/api/auth/login` 无需认证
- **其余所有 `/api/*`**: 强制JWT验证，401返回中文提示
