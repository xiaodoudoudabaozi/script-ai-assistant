## 一、技术栈

```
浏览器 (PWA)
    │  http://localhost:3009
    ▼
Next.js 14 (TypeScript)        FastAPI + RapidOCR       PostgreSQL 16      DeepSeek API
├─ 8个前端页面                ├─ DOCX/PDF/图片解析       ├─ 11张表           ├─ v4-pro (问答)
```

## 四、认证架构

```
请求 → middleware.ts (jose JWT验证)
     → 白名单 /api/auth/login: 放行
     → 其他 /api/*: 验证 session_token Cookie
     → 注入 x-user-id / x-user-role header
     → 路由内 getUser() 读取 → 权限判断
```

- 前端: 0处 `x-user-data` header（全部靠Cookie）
- 后端: 统一 `@/lib/auth` 的 `getUser()` / `isAdminOrLeader()`
- 异常追踪: 所有 `.catch(() => {})` 已替换为 `console.warn`
