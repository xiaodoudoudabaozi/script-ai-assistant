# 剧本杀AI店员助手 — 部署安装说明

> 版本：v1.0 | 适用于店内服务器/电脑，3 个 Docker 容器部署

---

## 1. 环境要求

| 项 | 最低 | 推荐 |
|---|------|------|
| CPU | 4 核 | 8 核+ |
| 内存 | 8 GB | 16 GB |
| 磁盘 | 50 GB 可用 | 100 GB+ |
| 系统 | Windows 10+ / Linux / macOS | Windows 11 / Ubuntu 22.04 |
| 网络 | 千兆局域网，固定 IP | 同左 |
| 软件 | Docker Desktop 4.x+ | 最新版 |

## 2. 安装 Docker

### Windows
1. 下载 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 安装时勾选 "Use WSL 2 instead of Hyper-V"
3. 重启电脑，启动 Docker Desktop
4. 打开终端（PowerShell 或 CMD），验证：`docker --version`

### Linux (Ubuntu)
```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
sudo systemctl enable docker
```

## 3. 项目文件准备

将项目目录 `claude_code` 完整复制到服务器，目录结构如下：

```
claude_code/
├── docker-compose.yml
├── .env                  # API Key 配置（需自行创建）
├── init-db.sql
├── fastapi-ai/
├── nextjs/
└── data/scripts/         # 剧本文件存储（自动创建）
```

## 4. 配置 API Key

在项目根目录创建 `.env` 文件（与 `docker-compose.yml` 同级）：

```
DEEPSEEK_API_KEY=sk-你的DeepSeek_API_Key
```

> DeepSeek API Key 获取：注册 [platform.deepseek.com](https://platform.deepseek.com)，进入「API Keys」创建。

## 5. 启动服务

```bash
cd claude_code
docker compose up -d
```

首次启动会自动：
- 拉取 PostgreSQL 镜像
- 构建 Next.js 和 FastAPI 镜像
- 初始化数据库表结构（`init-db.sql`）
- 创建数据卷（`pgdata`、`script-data`）

验证启动成功：

```bash
docker ps
# 应看到 3 个容器均为 Up 状态：nextjs、postgres、fastapi-ai
```

## 6. 访问地址

- 本机访问：`http://localhost:3000`
- 局域网访问：`http://192.168.x.x:3000`（替换为服务器实际 IP）

> 查看服务器 IP：Windows 终端输入 `ipconfig`，找到「IPv4 地址」。

## 7. 首次使用

### 7.1 创建管理员账号

1. 浏览器打开 `http://localhost:3000`
2. 目前还没有用户——需要通过数据库直接创建管理员：

```bash
# 进入容器生成 bcrypt 密码哈希
docker exec claude_code-nextjs-1 node -e "const bcrypt = require('bcrypt'); bcrypt.hash('你的密码', 10).then(h => console.log(h));"

# 复制输出的哈希值，然后插入管理员账号
docker exec -it claude_code-postgres-1 psql -U postgres -d scriptstore -c "
INSERT INTO employees (name, role, phone, position, password_hash)
VALUES ('管理员', 'admin', '13800000000', '店长', '粘贴上面生成的哈希值');
"
```

3. 刷新页面，用工号 `管理员` 和刚才设置的密码登录

### 7.2 上传剧本

1. 以管理员身份登录后，点击导航栏「剧本」进入剧本管理页
2. 点击「+ 添加剧本」
3. 填写剧本名称、类型、人数、难度等元数据
4. 拖拽或点击上传 DOCX/PDF 文件（最大 50MB）
5. 点击「保存」——剧本会被自动解析并缓存

### 7.3 创建员工账号

1. 管理员点击导航栏「员工」→「添加员工」
2. 填写姓名、电话、职位、密码
3. 员工角色选 `staff`（普通员工只能查自己的排班）

## 8. 手机端访问

### 8.1 配置固定 IP（推荐）

在路由器管理页面将服务器 MAC 地址绑定固定 IP，如 `192.168.1.100`。

员工手机浏览器访问 `http://192.168.1.100:3000` 即可。

### 8.2 HTTPS 自签名证书（可选，提升安全性）

```bash
# 安装 mkcert
choco install mkcert    # Windows
# 或 brew install mkcert  # macOS
# 或 apt install mkcert   # Linux

# 生成证书
mkcert -install
mkcert localhost 192.168.1.100

# 将生成的 .pem 文件配置到 Next.js
```

> v1 阶段内网 HTTP 访问即可满足安全需求，密码已在传输前 bcrypt 加密。

### 8.3 添加到手机主屏幕（PWA）

1. iPhone Safari：打开网址 → 底部「分享」→「添加到主屏幕」
2. Android Chrome：打开网址 → 菜单「添加到主屏幕」

## 9. 日常运维

### 9.1 常用命令

```bash
# 查看运行状态
docker compose ps

# 查看日志
docker compose logs -f nextjs       # Next.js 日志
docker compose logs -f postgres     # 数据库日志
docker compose logs -f fastapi-ai   # 解析服务日志

# 重启服务
docker compose restart nextjs

# 停止全部服务
docker compose down

# 启动全部服务
docker compose up -d

# 更新代码后重建
docker compose build nextjs
docker compose up -d nextjs
```

### 9.2 数据备份

```bash
# 每日自动备份（添加到 crontab / 任务计划）
# 备份数据库
docker exec claude_code-postgres-1 pg_dump -U postgres scriptstore > backup_$(date +%Y%m%d).sql

# 备份剧本文件（直接复制 data/scripts/ 目录）
cp -r data/scripts/ scripts_backup_$(date +%Y%m%d)/

# 保留最近 30 天备份，定期清理旧文件
```

### 9.3 磁盘空间监控

管理员可访问管理后台，仪表盘显示磁盘使用率：
- 80% 时告警提示
- 95% 时阻止新剧本上传

### 9.4 模型切换

当 DeepSeek v4-pro 出现故障时：
1. 管理员登录 → 系统设置（PC 端）
2. 将激活模型从 `deepseek-v4-pro` 切换为 `deepseek-v4-flash`
3. 实时生效，前端会显示当前模型名称

## 10. 常见问题

| 问题 | 解决方案 |
|------|---------|
| 端口 3000 被占用 | `netstat -ano \| findstr :3000` 查找占用进程并终止 |
| Docker 启动失败 | 检查 Docker Desktop 是否运行，WSL2 是否正常 |
| 上传剧本后无法问答 | 检查 DeepSeek API Key 是否正确配置，查看 `docker compose logs nextjs` |
| FastAPI 解析失败 | 确认文件为标准 DOCX/PDF，非加密/扫描版。查看 `docker compose logs fastapi-ai` |
| 手机无法访问 | 检查是否在同一局域网，防火墙是否放行 3000 端口 |
| 页面显示异常 | 清除浏览器缓存，注销 Service Worker（DevTools → Application → Service Workers → Unregister） |
