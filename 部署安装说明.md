## 1. 你需要准备什么

### 硬件
- 一台电脑（做服务器用，平时不关机）
- 建议 8GB 以上内存，50GB 以上剩余磁盘空间

### 软件
- **Windows 10 或 Windows 11**（本教程以 Windows 11 为例）
- 稳定的网络连接

### 时间
- 首次安装约 30-60 分钟（大部分时间在等下载）

## 2. 安装 Docker Desktop

Docker 是一个"容器"工具，可以一键启动整个项目。把它想象成一个虚拟机就行。

### 2.1 下载

打开浏览器，访问：
```
https://www.docker.com/products/docker-desktop/
```

点击页面上的 **Download for Windows** 按钮。文件大约 500MB。

### 2.2 安装

1. 双击下载的 `Docker Desktop Installer.exe`
2. 安装界面全部默认选项即可，一路点 **OK** / **Next**
3. 安装完成后会提示重启电脑，点 **重启**
4. 重启后 Docker Desktop 会自动启动（右下角任务栏会出现鲸鱼图标）
5. 如果弹出一个协议窗口，点 **Accept**（接受）

### 2.3 验证安装成功

按键盘上的 **Win 键**，输入 `PowerShell`，点击打开。

在打开的蓝色/黑色窗口里输入以下命令，然后按 **回车**：

```powershell
docker --version
```

如果看到类似 `Docker version 28.x.x` 的文字，说明安装成功。

> **小提示**：从现在开始，所有命令都在这个 PowerShell 窗口里输入，每次输入完按回车执行。

## 3. 获取项目文件

### 3.1 把项目文件夹放到电脑上

把 `claude_code` 文件夹整个复制到服务器的 **桌面** 上。

最终路径应该是：
```
C:\Users\你的用户名\Desktop\claude_code
```

为了教程方便，我们假设你的用户名是 `Tom`，路径为 `C:\Users\Tom\Desktop\claude_code`。如果用户名不同，请替换成你自己的。

### 3.2 验证文件完整

在 PowerShell 里输入：

```powershell
dir C:\Users\Tom\Desktop\claude_code
```

你应该能看到这些文件和文件夹：

```
docker-compose.yml
init-db.sql
fastapi-ai
nextjs
手册
.env
```

> 如果没有 `.env` 文件，不要急，下一步会创建。

## 4. 配置 API 密钥

这个项目使用 DeepSeek（深度求索）的 AI 服务，需要一个 API 密钥。

### 4.1 获取 DeepSeek API 密钥

1. 打开浏览器，访问：`https://platform.deepseek.com`
2. 用手机号注册账号（需要接收验证码）
3. 登录后点击左侧菜单「API Keys」
4. 点击「创建 API Key」按钮
5. 给密钥起个名字（比如"店内助手"），点确定
6. 页面会显示一串类似 `sk-xxxxxxxxxxxxxxxxxxxxxxxx` 的字符，**立刻复制下来**
7. **重要**：关闭页面后这串字符就看不到了，请务必保存好

> 费用：DeepSeek 按使用量收费，店内每天约 100 次问答，月费大约 60-70 元。

### 4.2 创建 .env 配置文件

在桌面上找到 `claude_code` 文件夹，双击打开。

在文件夹空白处 **右键 → 新建 → 文本文档**。

文件名改为 `.env`（注意：只有后缀，没有文件名，Windows 可能会警告你，点"是"）。

> 如果 Windows 不让你改名：在文件夹里点"查看"→ 勾选"文件扩展名"，然后把"新建文本文档.txt"改成".env"。

右键 `.env` 文件 → 打开方式 → 记事本。

在文件里输入以下内容（把 `sk-xxxxx` 替换成你刚才复制的真实密钥）：

```
DEEPSEEK_API_KEY=sk-你的密钥粘贴在这里
```

保存并关闭。

### 4.3 验证配置

在 PowerShell 里输入：

```powershell
type C:\Users\Tom\Desktop\claude_code\.env
```

应该能看到你刚才输入的内容。如果出错或为空，回到上一步检查。

## 5. 启动服务

### 5.1 切换到项目目录

在 PowerShell 输入：

```powershell
cd C:\Users\Tom\Desktop\claude_code
```

光标前面的路径会变成项目路径，说明切换成功。

### 5.2 启动（首次需要较长时间）

```powershell
docker compose up -d
```

按回车后，Docker 会开始下载和构建。**这个过程需要 5-20 分钟**，取决于网速。你会看到很多文字在滚动，这是正常的。

看到类似下面的内容，说明启动完成：

```
Container claude_code-postgres-1  Healthy
Container claude_code-fastapi-ai-1  Started
Container claude_code-nextjs-1  Started
```

### 5.3 验证启动成功

输入以下命令查看运行状态：

```powershell
docker ps
```

应该看到 3 个容器都是 `Up` 状态：
- `claude_code-nextjs-1`（网站服务）
- `claude_code-postgres-1`（数据库）
- `claude_code-fastapi-ai-1`（文档解析）

## 6. 创建管理员账号

### 6.1 生成加密密码

在 PowerShell 输入（把 `你的密码` 改成你想用的密码，比如 `admin123`）：

```powershell
docker exec claude_code-nextjs-1 node -e "const bcrypt = require('bcrypt'); bcrypt.hash('你的密码', 10).then(h => console.log(h));"
```

按回车后，会输出一串类似 `$2b$10$UKL3ssq4Jp9...` 的字符。**用鼠标选中这串字符，右键复制**。

> 这串字符是你的密码的"加密版本"，数据库里存在的是加密后的，这样即使数据泄露别人也看不到你的真实密码。

### 6.2 写入管理员账号

复制下面这条命令到记事本，把 `粘贴刚才的哈希值` 替换成你上一步复制的那串字符，然后整条复制到 PowerShell 执行：

```powershell
docker exec claude_code-postgres-1 psql -U postgres -d scriptstore -c "INSERT INTO employees (name, role, phone, position, password_hash) VALUES ('管理员', 'admin', '13800000000', '店长', '粘贴刚才的哈希值');"
```

按回车后，如果看到 `INSERT 0 1`，说明创建成功。

> 如果提示 `ERROR: duplicate key`，说明管理员已经存在了，不用管，继续下一步。

### 6.3 验证账号

```powershell
docker exec claude_code-postgres-1 psql -U postgres -d scriptstore -c "SELECT name, role FROM employees;"
```

应该看到一行 `管理员 | admin`。

## 7. 登录并使用

### 7.1 打开网站

打开浏览器（推荐 Chrome 或 Edge），在地址栏输入：

```
http://localhost:3009
```

你会看到一个登录页面。

### 7.2 登录

- 工号/手机号：输入 `管理员` 或 `13800000000`
- 密码：输入你在第 6.1 步设置的密码

点击"登录"。

### 7.3 上传剧本

1. 登录后，点击页面顶部导航栏的「剧本」
2. 点击「+ 添加剧本」按钮
3. 填写剧本名称，可选填类型、人数、幕数等
4. 把剧本文件（支持 DOCX、PDF、PNG、JPG）拖进虚线框
5. 如果不确定文件类型，点「🤖 AI 自动分类」让 AI 帮你分
6. 点击「保存」按钮
7. 弹窗自动关闭，文件在后台解析，卡片上会显示解析进度（如"15/40 已解析"）

### 7.4 提问剧本内容

1. 回到首页，点击左上角「+ 选择剧本」
2. 选择你刚才上传的剧本
3. 在输入框输入问题，比如"凶手是谁？""关键线索有哪些？"
4. 也可以直接点卡片上的快捷提问按钮

### 7.5 创建员工账号

1. 点击导航栏「员工」
2. 点击「添加员工」
3. 填写姓名、电话、职位、密码
4. 保存后员工就可以登录了

## 8. 让其他设备也能访问

### 8.1 查看服务器 IP 地址

在服务器电脑的 PowerShell 输入：

```powershell
ipconfig
```

找到你正在使用的网络适配器（通常叫"以太网适配器"或"无线局域网适配器 Wi-Fi"），下面有一行 `IPv4 地址`，通常是 `192.168.x.x` 格式。

记下这个地址，比如 `192.168.1.100`。

### 8.2 员工手机/电脑访问

在员工设备的浏览器地址栏输入：

```
http://192.168.1.100:3009
```

（把 `192.168.1.100` 换成你刚才查到的真实 IP）

> 如果无法访问，检查：
> 1. 是否连接了同一个 WiFi
> 2. 防火墙是否拦截了 3009 端口（见第 10 节）

### 8.3 添加到手机主屏幕

- **iPhone**：Safari 打开网址 → 底部"分享"图标 → "添加到主屏幕"
- **Android**：Chrome 打开网址 → 右上角"..."菜单 → "添加到主屏幕"

添加后就像普通 App 一样使用了，离线时也能显示缓存页面。

## 9. 日常维护

### 9.1 启动和停止

每次电脑重启后，Docker Desktop 会自动启动。但如果容器没起来，打开 PowerShell 执行：

```powershell
cd C:\Users\Tom\Desktop\claude_code
docker compose up -d
```

要停止服务（比如电脑要关机）：
```powershell
cd C:\Users\Tom\Desktop\claude_code
docker compose down
```

### 9.2 数据备份（建议每周做一次）

在 PowerShell 里执行以下命令。每行输入后按回车：

```powershell
# 切换到桌面
cd C:\Users\Tom\Desktop

# 备份数据库（把今天日期替换成实际日期，比如 20260501）
docker exec claude_code-postgres-1 pg_dump -U postgres scriptstore > 剧本数据库备份_$(Get-Date -Format 'yyyyMMdd').sql

# 备份剧本文件
Copy-Item -Recurse C:\Users\Tom\Desktop\claude_code\nextjs\data\scripts 剧本文件备份_$(Get-Date -Format 'yyyyMMdd')
```

备份文件会出现在桌面上。建议把这些备份文件复制到 U 盘或网盘。

### 9.3 恢复数据

如果需要从备份恢复数据库：

```powershell
# 先确保容器在运行
cd C:\Users\Tom\Desktop\claude_code
docker compose up -d

# 恢复数据库（把备份文件名改成实际的）
type 剧本数据库备份_20260501.sql | docker exec -i claude_code-postgres-1 psql -U postgres -d scriptstore
```

### 9.4 更新代码

当你拿到新版本的项目文件夹后：

```powershell
cd C:\Users\Tom\Desktop\claude_code
docker compose down
# 用新的 claude_code 文件夹覆盖旧的
docker compose build --no-cache
docker compose up -d
```

## 10. 常见问题处理

### 问题：Docker Desktop 启动后一直转圈

**原因**：WSL2 内核需要更新。

**解决**：
1. 打开 PowerShell，输入：`wsl --update`
2. 等待完成后重启电脑
3. 再次启动 Docker Desktop

### 问题：端口 3009 被占用

**现象**：启动时提示 `port 3009 already in use`

**解决**：在 PowerShell 里找到并关闭占用端口 3009 的程序：
```powershell
netstat -ano | findstr :3009
```
记下最后一列的数字（PID），然后：
```powershell
taskkill /PID 那个数字 /F
```
再重新执行 `docker compose up -d`。

### 问题：手机无法访问

**检查步骤**：
1. 手机和服务器是否连接了同一个 WiFi
2. 在服务器 PowerShell 输入 `ipconfig`，确认 IP 地址是否还是之前那个（可能会变）
3. 检查 Windows 防火墙：
   - 按 Win 键，输入"防火墙"，打开"Windows Defender 防火墙"
   - 点左侧"高级设置"
   - 点左侧"入站规则"→ 右侧"新建规则"
   - 选择"端口"→ TCP → 特定本地端口输入 `3009`
   - 选择"允许连接"→ 全部勾选 → 名称填"AI店员助手"
   - 完成

### 问题：上传剧本后问不出内容

**检查步骤**：
1. 确认 `.env` 文件里的 `DEEPSEEK_API_KEY` 是正确的
2. 确认 DeepSeek 平台账号余额充足
3. 在 PowerShell 输入 `docker compose logs nextjs` 查看错误日志

### 问题：页面显示异常或按钮不工作

**解决**：
1. 按 Ctrl+Shift+Delete 清除浏览器缓存
2. 按 F12 → Application → Service Workers → 点 Unregister
3. 刷新页面（Ctrl+F5）
4. 重新登录

### 问题：Docker 占用磁盘太大

**解决**：定期清理 Docker 缓存
```powershell
docker system prune -a
```
出现提示时输入 `y` 确认。这不会影响你的数据。

### 问题：电脑关机后服务没启动

**解决**：Docker Desktop 默认开机自启，但容器需要手动启动。每次开机后：

1. 确认 Docker Desktop 右下角图标显示正常（不是黄色警告）
2. 打开 PowerShell 执行：
   ```powershell
   cd C:\Users\Tom\Desktop\claude_code
   docker compose up -d
   ```
